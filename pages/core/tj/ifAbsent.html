# IfAbsent (1.4.9+)

xxIfAbsent会自动判断条件值是否为空，且只会在不为空的情况下生效

# API

  • eq -> eqIfAbsent
  • ne -> neIfAbsent
  • gt -> gtIfAbsent
  • ge -> geIfAbsent
  • lt -> ltIfAbsent
  • le -> leIfAbsent
  • like -> likeIfAbsent
  • notLike -> notLikeIfAbsent
  • likeLeft -> likeLeftIfAbsent
  • likeRight -> likeRightIfAbsent
  • notLikeLeft -> notLikeLeftIfAbsent
  • notLikeRight -> notLikeRightIfAbsent

举例:

wrapper.eq(Objects.nonNull(name), UserDO::getName, name)

等效于

wrapper.eqIfAbsent(UserDO::getName, name)

# 判断策略

  • not_empty 默认
    如果值类型为String则调用 StringUtils.isNotEmpty (opens new window)
    其他数据类型调用 Objects.nonNull

  • not_null
    全部调用 Objects.nonNull

  • not_blank
    如果值类型为String则调用 StringUtils.isNotBlank (opens new window)
    其他数据类型调用 Objects.nonNull

# 配置文件

mybatis-plus-join:
  #xxIfAbsent判断策略 not_null/not_empty/not_blank
  if-absent: not_empty

# 自定义Wrapper策略

//针对指定wrapper生效
wrapper.setIfAbsent(val -> null != val);
//重载，enums参数用法同下
wrapper.setIfAbsent((val, enums) -> null != val);

# 自定义全局策略

@Configuration
public class MybatisPlusConfig {
    /**
     * 自定义ifAbsent策略 优先级高于配置文件
     * 只对 xxIfAbsent生效 比如eqIfAbsent、neIfAbsent等
     */
    @Bean
    public MybatisPlusJoinIfAbsentConsumer mybatisPlusJoinIfAbsentConsumer() {
        // val是条件值 enums是条件类型
        return (val, enums) -> null != val;
    }
